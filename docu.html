Spiel Vorschau

Hier sieht man, wie unser Spiel ungefähr aufgebaut ist. Oben stehen die beiden Schwierigkeitsgrade Easy und Hard. Darunter wird eine Flagge angezeigt und darunter gibt es vier Antwortmöglichkeiten. Oben links steht der Highscore und oben rechts die Anzahl richtiger Antworten. Im Hard-Modus gibt es zusätzlich einen Timer von 10 Sekunden.

Highscore: 0
Richtig: 0
Zeit: 10s

In Easy gibt es vier Antwortmöglichkeiten. In Hard kommt zusätzlich der Timer dazu.

So funktioniert das Spiel

Im Spiel wird dem Spieler eine Flagge angezeigt. Die Aufgabe ist es, das richtige Land zu erkennen.

Unter der Flagge gibt es vier Auswahlmöglichkeiten. Nur eine Antwort ist korrekt.

Nach jeder Antwort bekommt man direkt eine Rückmeldung. Es gibt zwei Level: Easy und Hard. Bei Easy hat man vier Auswahlmöglichkeiten und keinen Zeitdruck. Bei Hard gibt es zusätzlich einen Timer von 10 Sekunden, der das Spiel schwieriger macht.

Unsere Idee

Am Anfang hatten wir verschiedene Ideen wie Pong oder Tic Tac Toe.

Diese Spiele waren aber entweder zu kompliziert oder schon vergeben.

Deshalb haben wir uns für ein Flaggen-Quiz entschieden, da es einfacher umzusetzen ist und trotzdem ein interessantes Spiel ergibt.

Code Erklärung

Die Datei quiz.js steuert das ganze Verhalten unseres Spiels. Mit diesem Code wollten wir erreichen, dass das Flaggen-Quiz funktioniert und auf die Eingaben des Spielers reagiert.

Am Anfang holt der Code wichtige Elemente aus der HTML-Datei mit document.getElementById(...). Dazu gehören das Bild der Flagge, die vier Antwort-Buttons, der Button für die nächste Frage sowie die Anzeige für den Punktestand und den Highscore. Dadurch kann JavaScript später diese Elemente verändern und das Spiel steuern.

Danach werden die vier Antwort-Buttons in einem Array gespeichert. Das macht den Code einfacher, weil man dann mit einer Schleife über alle Buttons gehen kann. So können zum Beispiel alle Buttons gleichzeitig deaktiviert, zurückgesetzt oder neu beschriftet werden.

Ausserdem gibt es wichtige Variablen wie flags, answer, score, highscore, difficulty und timeLeft. In ihnen speichert das Spiel seinen aktuellen Zustand. So weiss das Programm immer, welche Flagge gerade angezeigt wird, welche Antwort richtig ist, wie viele Punkte der Spieler hat und ob Easy oder Hard ausgewählt wurde.

Der Code erstellt auch neue Elemente direkt mit JavaScript. Dazu gehören der Timer und die Auswahl für die Schwierigkeitsgrade Easy und Hard. Diese Elemente werden vor dem Bild der Flagge eingefügt, damit zuerst die Auswahl erscheint und danach das eigentliche Spiel beginnt.

Eine wichtige Funktion ist updateScore(). Sie sorgt dafür, dass die Anzeige für den aktuellen Punktestand und den Highscore immer aktualisiert wird. Ohne diese Funktion würde sich die Anzeige nicht verändern, obwohl im Hintergrund neue Werte gespeichert werden.

Mit der Funktion randomIndex() wird eine Zufallszahl erzeugt. Diese wird gebraucht, um zufällig eine Flagge aus der Datei flags.json auszuwählen. So ist jede Runde anders und das Spiel bleibt abwechslungsreich.

Die Funktion getFileName() holt aus einem Bildpfad nur den eigentlichen Dateinamen heraus. Danach wird mit setImageFromFlag() das passende Bild der Flagge gesetzt. So erscheint immer das richtige Bild zur aktuellen Frage.

Mit showCorrectAnswer() wird die richtige Antwort grün markiert. Das ist wichtig, damit der Spieler nach dem Klicken sofort sieht, welche Lösung korrekt war. Bei einer falschen Antwort wird zusätzlich der gewählte Button rot.

Die Funktionen hideQuiz() und showQuiz() regeln, was am Anfang sichtbar ist. Zuerst soll nicht sofort das ganze Spiel erscheinen, sondern zuerst die Auswahl zwischen Easy und Hard. Erst nach dieser Auswahl wird das eigentliche Quiz eingeblendet.

Im Hard-Modus wird ein Timer gestartet. Die Funktion startTimer() setzt die Zeit auf 10 Sekunden und zählt dann jede Sekunde herunter. Wenn die Zeit abläuft, wird die Funktion handleTimeout() aufgerufen. Dann werden die Buttons deaktiviert, die richtige Antwort wird angezeigt und der aktuelle Punktestand wird auf null gesetzt. Der Highscore bleibt aber erhalten.

Die Funktion setQuestion() erstellt eine neue Frage. Dabei wird zuerst eine zufällige Flagge ausgewählt. Danach wird die richtige Antwort gespeichert und drei weitere falsche Antworten werden ergänzt. Anschliessend werden die vier Antworten gemischt, damit die richtige Lösung nicht immer an derselben Stelle steht. Dann werden die Texte der Buttons neu gesetzt und der Timer startet erneut.

Wenn der Spieler auf eine Antwort klickt, wird die Funktion selectOption() ausgeführt. Sie überprüft, ob die angeklickte Antwort mit der richtigen Lösung übereinstimmt. Ist die Antwort richtig, wird der Punktestand erhöht. Wenn der neue Wert grösser ist als der bisherige Highscore, wird auch der Highscore angepasst. Bei einer falschen Antwort wird der aktuelle Punktestand wieder auf null gesetzt.

Damit das Spiel überhaupt Fragen stellen kann, müssen zuerst die Flaggendaten geladen werden. Das macht die Funktion loadFlags(). Sie lädt die Datei flags.json, liest die enthaltenen Daten ein und speichert nur die gültigen Einträge. Dadurch stehen dem Spiel alle Flaggen zur Verfügung.

Die Funktion startGame() startet das Spiel, nachdem der Spieler Easy oder Hard gewählt hat. Dabei wird der Schwierigkeitsgrad gespeichert, die Auswahl wird ausgeblendet, das Quiz wird sichtbar gemacht und die erste Frage wird geladen.

Ganz am Schluss gibt es noch die Funktion initQuiz(). Sie bereitet das ganze Spiel vor. Dort werden die Flaggendaten geladen, die Buttons mit den richtigen Funktionen verbunden und das Quiz zuerst versteckt. Danach wird der Startzustand angezeigt. Diese Funktion ist also die Startfunktion des ganzen Programms.

Zusammengefasst sorgt quiz.js dafür, dass aus der HTML-Struktur ein interaktives Spiel wird. Der Code lädt Flaggen, zeigt Fragen an, überprüft Antworten, zählt Punkte, speichert den Highscore und ergänzt im Hard-Modus zusätzlich einen Timer von 10 Sekunden.
